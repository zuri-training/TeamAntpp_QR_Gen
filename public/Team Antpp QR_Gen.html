Research Background

What is QR code ?

QR Code is a two-dimensional version of the barcode, typically made up of black and white pixel patterns. Denso Wave, a Japanese subsidiary of the Toyota supplier Denso, developed them for marking components in order to accelerate logistics processes for their automobile production. Now, it has found its way into mobile marketing with the widespread adoption of smartphones. "QR" stands for "Quick Response", which refers to the instant access to the information hidden in the Code.

Also, there are two types of QR codes; static and dynamic QR codes. Static QR codes cannot be altered upon creation, while dynamic QR codes are easy to change and rearrange. It means that you can change the content and reuse code as many times as you like.

What are the benefits of using QR Codes?

They are gaining popularity because of their versatility. You can use them to gather feedback to improve your products or services, increase customer engagement with images or videos, or even promote your business via events and coupons. All of these can be done with just a single scan!

How do I scan QR Codes?

Depending on your device, you might already have a built-in QR Code reader or scanner. Open the camera app on your mobile phone and hold it over a Code for a few seconds until a notification pops up.

Problem Statement

Most people do not know how to use QR code related platforms. QR Codes have proven to be useful in sharing a large amount of information with multiple people through a single share however, a lot of people do not know how to come about generating them. Also, a lot of the online QR code generators are complicated and not easy to use/ beginner-friendly.

DATABASE SCHEMA DESIGN

 

The database schema design has four tables. The relational database management system for this project is MySQL and the database schema design is done using dbdiagram.io. The figure below shows the database schema

 

 

The tables in the database are users, qrcode, type and file. The descriptions of the tables and the information they hold are:

 

Users

Users table holds the users information. The users table has the login details that enables users to sign up, log in and manage their QR-Go accounts. It also has an optional signature that can be added to the QR file. This signature will be added to each QR code generated by that user.

 

Qrcode

The qrcode table holds information about each qrcode that is generated. It has columns such as user_id which is a foreign_key linking the qrcode to the user account that owns that code. It also has a file_id which is also a foreign key to the file table, linking each generated qrcode to the file generated from the code. These are the files that will be downloaded to the device or share to social media platforms.

 

Type

The third table is the type table. This is basically a look up table that contains information about the type of file being generated.

 

File

The last table is the file table. It is linked to the qrcode code table via the file_id column. It stores each file being generated from the qrcode.
